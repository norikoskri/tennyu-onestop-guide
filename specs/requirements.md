# 要件定義

**プロジェクト名**: 地域ガイド（転入者向け地域情報プラットフォーム）
**作成日**: 2026-01-19
**対象ユーザー**: 転入者（新しい地域へ引越しする人）

---

## 📋 プロジェクト概要

転入者が新しい地域の情報を取得し、引越しに必要な手続きを一気通貫で支援するWebアプリケーション。

### 目的
- 転入者が「地域の暗黙知」を簡単に入手できるようにする
- 最小の入力で最大の手続き支援を実現する
- 手続きの漏れや遅延を防ぎ、転入者の負担を軽減する

### 設計思想
- **最小の入力で最大の支援**: 1回の情報入力で一気通貫の支援を提供
- **実行できるところは自動、最後はユーザーが確定**: 本人確認・電子署名が必要な部分はユーザーの最終確認に委ねる
- **AIエージェントによる負荷軽減**: 入力・タスク管理・意思決定をAIが支援し、ユーザーの認知負荷を最小化

## 📮 JPDオリジナル地域情報コンテンツ

### 郵便局員由来の情報とは

郵便局員は毎日地域を巡回し、住民と直接やり取りする中で「公式情報には載っていない地域の実態」を把握しています。

この**現場知**をコンテンツ化することで、他サービスにはない独自価値を提供します。

### 情報カテゴリと郵便局員ならではの視点

| カテゴリ | 公式情報（自治体HP等） | 郵便局員ならではの情報（現場知） |
|---------|----------------------|------------------------------|
| **ゴミ出し** | 曜日・分別ルール | 「この集積所は朝7時までに出さないと回収されない」「年末年始の収集スケジュールが変わる」 |
| **買い物** | 店舗一覧・営業時間 | 「〇〇スーパーは火曜が特売日」「△△は品揃えが良い」「□□は駐車場が広い」 |
| **医療** | 病院リスト | 「□□医院は予約なしでも待ち時間が短い」「小児科は◎◎が評判」「〇〇薬局は在庫が豊富」 |
| **町内会** | 連絡先・会費 | 「会長の〇〇さんは話しやすい」「回覧板は△△さんに渡す」「イベント参加は任意」 |
| **防災** | 避難所一覧・ハザードマップ | 「この道は大雨で浸水しやすいので避けた方がいい」「〇〇公園は高台で安全」 |
| **子育て** | 保育園リスト・学校情報 | 「〇〇公園は午前中ママさんが集まる」「△△児童館のイベントが人気」「□□保育園は延長保育が充実」 |
| **グルメ** | 飲食店リスト | 「〇〇のランチはボリューム満点」「△△は地元民に愛される老舗」「□□は子連れでも入りやすい」 |
| **交通** | 路線図・時刻表 | 「朝8時台の〇〇駅は混雑する」「△△バス停は屋根がないので雨の日注意」 |

### 情報の信頼性表示

各情報には情報源を明示し、ユーザーが信頼性を判断できるようにします。

| 情報源タイプ | 表示ラベル | 説明 |
|-------------|-----------|------|
| **郵便局員** | 📮 配達員からの情報 | 日々の巡回で得た現場知・実体験 |
| **地域住民** | 👥 地元の方からの情報 | 住民から聞いた口コミ・評判 |
| **公式** | 🏛️ 公式情報 | 自治体HP・公式サイトからの引用 |

### 競合との差別化

| 競合サービス | 提供情報 | JPDの優位性 |
|-------------|---------|------------|
| **Google Maps** | 店舗・施設の場所、レビュー | 「行ってみた感想」「地元民のおすすめポイント」がない |
| **自治体HP** | 制度・手続き情報 | 「実際にやってみた時のコツ」「窓口の混雑状況」がない |
| **不動産サイト** | 物件周辺情報 | 契約後のフォローがない、情報が浅い |
| **口コミサイト** | ユーザーレビュー | 匿名で信頼性が不明、転入者向けに整理されていない |
| **JPD** | 現場知 + 一気通貫支援 | **「住んでみないとわからない」情報** + 手続き支援 |

### モックで検証する差別化仮説

> 「郵便局員が日々の配達で得た**現場知**は、Google Mapsや自治体HPにはない独自価値があり、転入者はその情報に価値を感じる」

**検証ポイント**:
- 公式情報だけでは得られない「実用的なコツ」に価値を感じるか
- 「郵便局員からの情報」という出典表示が信頼感を高めるか
- パーソナライズされた地域情報でサービスの継続利用意向が高まるか

## 🤖 AIエージェント設計方針

### 基本原則
ユーザーの**入力負荷**と**タスク管理負荷**をAIエージェントで最大限軽減する。

### AIが担う役割

| 領域 | AIの役割 | ユーザーのアクション |
|------|---------|---------------------|
| **情報収集** | 会話形式で必要情報を収集・構造化 | 質問に答えるだけ |
| **タスク生成** | 状況から必要な手続きを自動判定 | 確認のみ |
| **優先順位付け** | 期限・依存関係から最適な順序を提案 | 承認のみ |
| **次アクション提案** | 今やるべきことをプロアクティブに提示 | 選択するだけ |
| **質問応答** | 手続きに関する疑問に即時回答 | 質問するだけ |
| **進捗追跡** | 完了検知・リマインド・状況把握 | 報告するだけ |

### 入力軽減の具体策

| 手法 | 内容 | 削減効果 |
|------|------|---------|
| **会話形式入力** | フォームではなくチャットで情報収集 | 入力項目の認知負荷↓ |
| **郵便番号自動補完** | 郵便番号のみで住所・地域情報を取得 | 住所入力の手間↓ |
| **家族構成からの推定** | 「子供2人(3歳,7歳)」→ 必要手続きを自動判定 | 選択・判断の手間↓ |
| **段階的開示** | 必要な情報だけを必要な時に質問 | 一度に考える量↓ |

### タスク管理軽減の具体策

| 手法 | 内容 | 削減効果 |
|------|------|---------|
| **プロアクティブ提案** | 「次は〇〇をしましょう」と誘導 | 判断の手間↓ |
| **期限ベースの優先表示** | 急ぎのタスクを自動でハイライト | 確認の手間↓ |
| **完了の簡易報告** | 「完了しました」の一言で更新 | 操作の手間↓ |
| **コンテキスト保持** | 前回の続きから自動で再開 | 思い出す手間↓ |

## 👤 ユーザーストーリー

### US-0: AIアシスタントによる情報収集（入力負荷軽減）

**As a** 転入者
**I want** AIアシスタントとの会話で必要な情報を伝える
**So that** フォーム入力の手間を省き、何を入力すべきか考えなくて済む

#### 受け入れ基準
- [ ] チャット形式で基本情報を入力できる
- [ ] AIが質問を順番に投げかけ、必要な情報を収集する
- [ ] 郵便番号を入力すると住所が自動補完される
- [ ] 曖昧な入力（「来月引っ越します」等）をAIが解釈して確認する
- [ ] 入力内容の確認画面で一括修正できる
- [ ] 入力完了後、自動的にやることリストが生成される

#### 優先度
- P0（必須）

#### AIの振る舞い例
```
AI: こんにちは！引越しのお手伝いをします。
    まず、引越し先の郵便番号を教えてください。

User: 154-0001

AI: 東京都世田谷区池尻ですね。
    引越し予定日はいつ頃ですか？

User: 来月の15日くらい

AI: 2月15日頃ですね。
    ご家族は何人でお住まいになりますか？

User: 妻と子供2人です。子供は5歳と2歳

AI: ご家族4人（ご本人、奥様、5歳と2歳のお子様）ですね。
    お子様がいらっしゃるので、保育園・幼稚園の手続きも
    リストに追加しておきますね。

    [確認画面を表示]
```

### US-1: 基本情報入力

**As a** 転入者
**I want** 自分と家族の基本情報を1回だけ入力する
**So that** 以降の手続き支援で何度も同じ情報を入力しなくて済む

#### 受け入れ基準
- [ ] 転入者情報（氏名、生年月日、旧住所、新住所）を入力できる
- [ ] 家族構成（同居家族の情報）を入力できる
- [ ] 転入予定日を入力できる
- [ ] 入力した情報がローカルに保存される

#### 優先度
- P0（必須）

---

### US-2: やることリスト自動生成

**As a** 転入者
**I want** 基本情報を入力したら必要な手続きのリストが自動で生成される
**So that** 何をすべきか調べる手間が省ける

#### 受け入れ基準
- [ ] 入力情報から必要な手続きを自動判定して一覧表示される
- [ ] 手続きごとの期限・優先度が表示される
- [ ] 自動化レベル（案内のみ/予約支援/事前入力/オンライン申請）が明示される
- [ ] チェックリスト形式で表示される

#### 優先度
- P0（必須）

---

### US-3: 地域ガイド表示

**As a** 転入者
**I want** 転入先の地域情報を一覧で見る
**So that** 新しい地域の生活に必要な情報を把握できる

#### 受け入れ基準
- [ ] 住所（郵便番号）を入力すると、その地域の情報が表示される
- [ ] ゴミ出しルール（曜日、分別方法）が表示される
- [ ] 町内会・自治会情報が表示される
- [ ] おすすめ店舗（スーパー、飲食店、病院等）が表示される
- [ ] 地域行事・イベント情報が表示される
- [ ] 災害時の注意点・避難所情報が表示される
- [ ] 情報源（例: 郵便局員）が明示される

#### 優先度
- P0（必須）

---

### US-4: パーソナライズ表示

**As a** 転入者
**I want** 家族構成や関心事に応じて情報が優先表示される
**So that** 自分に必要な情報を効率よく見つけられる

#### 受け入れ基準
- [ ] プロフィール入力画面で家族構成を選択できる（単身/夫婦/子育て世帯/シニア等）
- [ ] 関心事を選択できる（子育て/医療/買い物/グルメ/スポーツ等）
- [ ] 子育て世帯の場合、保育園・学校・公園情報が優先表示される
- [ ] シニアの場合、病院・介護・買い物支援情報が優先表示される
- [ ] 「あなたへのおすすめ」セクションが表示される

#### 優先度
- P1（重要）

---

### US-5: 手続き詳細表示

**As a** 転入者
**I want** 各手続きの詳細情報（必要書類・窓口情報）を確認する
**So that** 手続きをスムーズに進められる

#### 受け入れ基準
- [ ] 手続きごとの必要書類一覧が表示される
- [ ] 窓口情報（名称、住所、営業時間）が表示される
- [ ] オンライン申請可能な場合はリンクが表示される
- [ ] 来庁必須の場合はその旨が明示される

#### 優先度
- P1（重要）

---

### US-6: 進捗管理ダッシュボード

**As a** 転入者
**I want** 手続きの完了状況を一覧で管理する
**So that** 何が終わって何が残っているかを把握できる

#### 受け入れ基準
- [ ] 手続きの完了状況を一覧表示できる
- [ ] 手続きを完了としてチェックオフできる
- [ ] 期限が近い手続きがハイライト表示される
- [ ] 全体の進捗率が表示される

#### 優先度
- P1（重要）

---

### US-7: 地域情報詳細

**As a** 転入者
**I want** 各カテゴリの詳細情報を閲覧する
**So that** 必要に応じて深い情報を得られる

#### 受け入れ基準
- [ ] カテゴリ一覧から詳細ページに遷移できる
- [ ] 各カテゴリ（生活基本、子育て、医療・健康、買い物、グルメ、行事・イベント、防災、交通）の詳細情報が表示される

#### 優先度
- P1（重要）

---

### US-8: AIによるプロアクティブ支援（タスク管理負荷軽減）

**As a** 転入者
**I want** AIが次にやるべきことを提案してくれる
**So that** 自分で考える・調べる・管理する負荷を減らせる

#### 受け入れ基準
- [ ] ダッシュボードで「次のおすすめアクション」が表示される
- [ ] 期限が近い手続きは自動でハイライト・通知される
- [ ] 手続きに関する質問にAIがその場で回答する
- [ ] 「完了しました」と報告するだけで進捗が更新される
- [ ] 前回の続きから自動で状況を把握して再開できる

#### 優先度
- P0（必須）

#### AIの振る舞い例
```
[ダッシュボード表示時]
AI: おかえりなさい！前回の続きですね。

    📌 今日のおすすめアクション:
    「転入届」の提出期限が3日後です。
    必要書類は準備できていますか？

User: 本人確認書類って何が必要？

AI: 転入届には以下のいずれかが必要です:
    ・マイナンバーカード
    ・運転免許証
    ・パスポート + 住民票

    マイナンバーカードをお持ちなら、
    それ1枚で大丈夫ですよ。

User: 転入届出してきた

AI: お疲れさまでした！「転入届」を完了にしますね ✓

    次は「マイナンバーカード住所変更」がおすすめです。
    転入届と同じ窓口でできますが、今日やりましたか？
```

### US-9: AIによる質問応答

**As a** 転入者
**I want** 手続きや地域に関する疑問をその場で質問できる
**So that** 自分で調べる手間を省ける

#### 受け入れ基準
- [ ] チャット画面で自由に質問できる
- [ ] 手続きの必要書類・期限・窓口情報について回答される
- [ ] 地域情報（ゴミ出し、病院、学校等）について回答される
- [ ] 回答の根拠（情報源）が明示される

#### 優先度
- P1（重要）

## 🚫 非機能要件

### パフォーマンス
- ページ読み込み: 3秒以内

### セキュリティ
- 個人情報はローカル（localStorage）のみに保存
- サーバーへの送信は行わない（モック版）

### 互換性
- 対応ブラウザ: Chrome, Safari, Firefox, Edge（最新版）
- デバイス: PC, タブレット, スマートフォン

### データ
- モックデータとして2〜3地域分を用意（例: 東京都世田谷区、神奈川県横浜市、福岡県福岡市）

## 🎯 成功指標

### 地域情報
- [ ] 住所を入力すると、その地域の情報が表示される
- [ ] 家族構成・関心事に応じて情報の優先順位が変わる
- [ ] 「郵便局員からの情報」という出典が明示されている

### 手続き支援
- [ ] 基本情報を1回入力すると、やることリストが自動生成される
- [ ] 各手続きの必要書類・窓口情報が表示される
- [ ] 手続きの完了状況を管理できる（チェックオフ）
- [ ] 自動化レベルが明示されている

### AIエージェント体験
- [ ] 会話形式で情報入力が完了できる（フォーム不要）
- [ ] AIが次のアクションを提案してくれる
- [ ] 質問に対してAIが即座に回答する
- [ ] 「完了」と言うだけで進捗が更新される

### 全体
- [ ] モバイル・PCで閲覧可能
- [ ] 「最小入力 → 最大支援」の体験が伝わる
- [ ] ユーザーが「考える・調べる・管理する」負荷が軽減されている
